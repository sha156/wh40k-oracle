## 为什么单独 spike

伤害分配核是全篇最高技术风险（spec 第七/十二节）：它要在**跨 N 向量化**的同时，正确表达四条互相纠缠的规则——
不溢出、变长 D、已损伤模型优先、逐点 FNP，外加**致命伤池**（dev wounds）的独立成池结算。
向量化写错不会崩，只会**悄悄算错期望伤害**，属于典型"自洽地错"。故在写 `sequence.py` 前先一次性对拍验证。

## 验证方法：双实现对拍（duipai）

用两套**相互独立**的实现互相印证，两者吃**同一份预掷好的输入数组**（伤害值 + 逐点 FNP 骰面），
从而把"分配逻辑"与"采样随机性"隔离，误差要求 = 逐元素**精确相等**（整数，非容差）：

| 实现 | 角色 | 特征 |
|------|------|------|
| `allocate_scalar` | oracle（真值） | 纯 Python 逐迭代逐攻击，规则显式、显然正确 = 手算的一般化 |
| `allocate_numpy` | 受测 + 复用 kernel | 在攻击槽维度小循环（≤ 数十），每槽跨 N 一次算完；**P4-b `sequence.py` 直接 lift** |

## 结果

**① 手算锚点（4 例，numpy vs 人手写死的期望值，逐项精确相等）**

| 锚点 | 覆盖规则 | 期望 (kills, eff, through, wiped) | 结果 |
|------|---------|--------------------------------|------|
| A 3×W3 / 四发 D2 / 无 FNP | 不溢出 + 已损伤优先（连打同一模型至死，每杀浪费 1） | (2, 6, 8, False) | PASS |
| B1 1×W1 / D6=6 / 无 FNP | 变长 D 单发不溢出（杀 1 浪费 5） | (1, 1, 6, True) | PASS |
| B2 1×W12 / 两发 D6=6 / 无 FNP | 多点在同一模型累加、不重置 | (1, 12, 12, True) | PASS |
| C 1×W6 / 两发 D6 / FNP5+ | 逐点 FNP 独立掷（发1 六点里 3 点免伤） | (1, 6, 9, True) | PASS |
| D 2×W3 / 正常1发+dev2份 / FNP6+ | 致命伤池正常伤后成池、接续被打伤模型、每份不溢出、FNP 逐点 | (1, 4, 5, False) | PASS |

**② 随机对拍**：400 场景 × N=200 = **80,000 次独立迭代，不一致元素 0**。
覆盖 M1-12 / W1-12（含单模型巨兽与虫群）/ 0-14 正常攻击 / 0-5 dev / 伤害 1-6 / FNP {None,4+,5+,6+}。

**③ 性能与内存**（N=10000，40 攻击槽 + 10 dev，最坏情形）：

- 三维 FNP 数组 (N, 50, 6) = 300 万 cell，int8 实占 **2.86 MB**（spec 担心的 (N,maxA,maxD) 内存 → 可接受）。
- 分配核耗时 **33.2 ms**（万次迭代，纯 numpy）→ **亚秒达标**。
- 逐点 FNP 三维数组只在**每槽瞬态** (N, Dmax) 出现，峰值工作内存远小于全量三维输入。

## 落进 P4-b 的规则语义（锁定，现行 dataslate）

1. **不溢出 = 已损伤优先 = 同一累积**：状态量 `rem`（当前模型剩 W）跨攻击槽携带，`rem<=0` 才计一杀并重置为 W；
   溢出（`through - rem`）作废。这一条累积同时实现了"单次伤害不流向下一模型"与"打伤的模型先打死"。
2. **逐点 FNP**：每点独立掷 D6，`骰 ≥ X` 免伤。对一发 D 值先数出穿透点数 `through`，再对模型施加——
   与"逐点施加、死了就停"在杀伤计数上等价（`through ≥ rem ⟺ 死`），可安全向量化。
3. **致命伤池（DEVASTATING WOUNDS，现行 dataslate）**：本单位正常伤害**全部结算后**成池；每个暴击造伤 = 一份 D 伤害，
   进池即无保存（保护在 wound/save 阶段已跳），**每份最多影响 1 个模型、溢出作废**（与正常伤害同样的不溢出），
   **FNP 仍逐点生效**。实现上 = 把 dev 份**追加在正常攻击槽之后**跑同一扫描（`rem` 状态自然携带）。
   ⚠ 这是"不溢出"的 dev wounds，**不是**会跨模型流的传统致命伤（后者是另一种 instance，本期不涉及；若 P5 需要传统 MW，另加一路）。
