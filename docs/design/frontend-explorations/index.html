各方向 steal / skip 速查

A · 军典页
  • Steal：A6 属性玉牌 + 被问武器整行高亮（任何方向通用）；A4 火漆印章判定（有明确裁决时记忆点极强）；A8 「P4 建设中」小字内嵌 CTA 的诚实写法；A9 chips 点击回填 + sticky 渐隐（交互骨架通用）。
  • Skip：A2 每条回答一张重卡片（多轮对话会笨重）；A1 羊皮纸书签牌页签（切换动效难做轻快）；A5 汉字序号圆印（条目上两位数就崩）；装饰整体（角花/双线）若走浅色方向全不可迁移。
B · 作战终端
  • Steal：B4 解算漏斗（命中→穿防→失防→伤害，全场最值得偷的部件，概率链问答的杀手锏）；B3 终端日志式轨迹的「降级=琥珀警示」语义；B1 状态栏放会话元信息的思路；B7 左人读右机读的引用分栏。
  • Skip：B9 闪烁光标和 >_ 提示符（非极客用户像 bug）；B2 英文呼号行（普通用户是噪音）；满页等宽中文混排（武器名+数字略挤）；1180px 以下 B4 已要降单列，窄屏成本高。
C · 数据工作台
  • Steal：C3 单行折叠轨迹+降级徽标常驻（透明度与干净的最优平衡）；C7 行内角标+底部来源表双形态引用；C4 判定胶囊+KPI 条；C1 侧栏放语境切换和会话历史；C6 overflow-x 容器保窄屏。
  • Skip：C2 完全文档流（对话感/人格感弱，像日志）；C6 纯缩写表头对新手是墙（需 tooltip 补救）；三处 sticky 叠加在 1024px 高度局促；全页无一处「40K 味」，气质上最平。
D · 战报专栏
  • Steal：D2 byline 元信息行（「检索 4 步 · 引用 3 条」一行字交代可信度）；D6 单位卡破栏加宽（密表不被正文栏勒死）；D5 关键词染色挂脚注（推导链和溯源焊在一起）；D4 pull-quote 判定的可读性。
  • Skip：D2 读者来信仪式感（多轮对话每轮重复会累赘）；D7 脚注要求来回滚动（边读边核对不友好）；D9 固定底栏 190px 占位（短回答时压阅读区）；报头静态感与「实时工具」定位有张力。
